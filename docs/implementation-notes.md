# 구현 메모

## 실제 자사몰에 붙일 위치

- 공통 레이아웃: `MarketingAutomation.init()`과 GTM 컨테이너 로드
- 동의 배너: `MarketingAutomation.setConsent()`
- 상품 상세 페이지: `trackViewItem()`
- 장바구니 버튼: `trackAddToCart()`
- 결제 시작 버튼 또는 결제 페이지 진입: `trackBeginCheckout()`
- 결제 성공 페이지: `trackPurchase()`
- 회원가입 완료: `trackSignUp()`
- 로그인 완료: `trackLogin()`
- 상담/쿠폰/문의 제출 완료: `trackGenerateLead()`

## 서버 책임

클라이언트 SDK는 새로고침 중복을 줄이지만 최종 방어선은 서버입니다.

- 결제 성공 페이지에는 실제 주문번호를 1개만 노출합니다.
- 같은 주문번호가 재렌더링되더라도 결제 성공 이벤트가 중복 발행되지 않게 서버 로그를 확인합니다.
- CRM 발송 서버는 수신거부와 마케팅 수신동의를 최종 검증합니다.
- CRM 수신 서버는 `automation_actions`를 계산해 발송툴로 전달합니다.
  - `add_to_cart`: 60분 뒤 장바구니 이탈 메시지, 구매 시 취소, 광고 대상 업데이트
  - `begin_checkout`: 30분 뒤 결제 이탈 메시지, 구매 시 취소, 광고 대상 업데이트
  - `purchase`: 7일 뒤 리뷰 요청, 30일 뒤 재구매 알림, 광고 리타겟팅 제외
  - `generate_lead`: 즉시 리드 후속 메시지
  - `dormant_60_days`, `dormant_90_days`: 즉시 휴면 복귀 메시지, 구매 시 제외되는 광고 대상 업데이트
  - `vip_qualified`: 즉시 VIP 혜택 메시지
- CRM 고객 집계 작업은 사용자별 휴면 60일, 휴면 90일, VIP 최초 충족 마일스톤을 각각 한 번만 발행합니다.
- 실제 발송툴은 예약 시각, 구매 취소 조건, 수신거부, 빈도 제한, 이벤트 중복 방지를 최종 집행합니다.

## 개인정보 원칙

- GA4/GTM dataLayer에는 이메일, 전화번호, 이름, 주소를 넣지 않습니다.
- CRM에는 마케팅 수신동의가 있는 연락처만 전달하며, 동의 값은 JSON 불리언 `true`만 인정합니다.
- 광고 전환 태그는 Consent Mode의 광고 동의가 있을 때만 실행합니다.

## 운영 지표

- 장바구니 이탈률
- 결제 이탈률
- 구매 전환율

- 구매 후 리뷰 작성률
- 재구매율
- 광고 전환 누락률
